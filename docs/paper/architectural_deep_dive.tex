\begin{abstract}
This document provides a comprehensive technical overview of the OpKAN architecture, a high-performance quantitative engine designed to solve the Heston Stochastic Volatility PDE. We introduce the Physics-Informed Kolmogorov-Arnold Network (PI-KAN) and detail how it leverages real-time LLM-driven topological mutations to outperform traditional Multilayer Perceptrons (MLPs) and numerical solvers in the context of high-order partial differential equations.

\section{Introduction}
The pricing of contingent claims under stochastic volatility, specifically the Heston model, remains a computationally demanding task. While Finite Difference Methods (FDM) and Monte Carlo (MC) simulations are industry standards, they suffer from the curse of dimensionality and rigidity in the face of regime shifts. OpKAN addresses these limitations by combining Physics-Informed Neural Networks (PINNs) with the Kolmogorov-Arnold representation theorem and LLM-based reasoning.

\section{The Mathematics of Heston PI-KAN}

\subsection{The Heston PDE}
The Heston model assumes that the underlying asset price $S_t$ and its variance $v_t$ follow coupled stochastic differential equations. The fair price of an option $V(S, v, t)$ must satisfy the following second-order parabolic PDE:
\begin{equation}
    \frac{\partial V}{\partial t} + \frac{1}{2}vS^2\frac{\partial^2 V}{\partial S^2} + \rho\sigma vS\frac{\partial^2 V}{\partial S\partial v} + \frac{1}{2}\sigma^2v\frac{\partial^2 V}{\partial v^2} + rS\frac{\partial V}{\partial S} + \kappa(\theta-v)\frac{\partial V}{\partial v} - rV = 0
\end{equation}
where:
\begin{itemize}
    \item $r$: Risk-free interest rate.
    \item $\kappa$: Speed of mean reversion of the variance.
    \item $\theta$: Long-term mean variance.
    \item $\sigma$: Volatility of volatility.
    \item $\rho$: Correlation between asset returns and variance.
\end{itemize}

\subsection{The Physics-Informed Loss Function}
The KAN is trained by minimizing a composite loss function $\mathcal{L}_{total}$ that penalizes deviations from the PDE residual and the boundary conditions (BC):
\begin{equation}
    \mathcal{L}_{total} = \mathcal{L}_{PDE} + \lambda_{BC} \mathcal{L}_{BC}
\end{equation}
The PDE residual loss $\mathcal{L}_{PDE}$ is defined over a set of collocation points in the interior domain $\Omega$:
\begin{equation}
    \mathcal{L}_{PDE} = \frac{1}{N} \sum_{i=1}^N \left| \mathcal{N}[V(S_i, v_i, t_i)] \right|^2
\end{equation}
where $\mathcal{N}$ is the differential operator representing the Heston PDE.

\subsection{Boundary Conditions}
To ensure a unique and financially consistent solution, we enforce:
\begin{enumerate}
    \item \textbf{Terminal Condition}: $V(S, v, T) = \max(S - K, 0)$ for a European Call.
    \item \textbf{Lower Boundary ($S \to 0$)}: $V(0, v, t) = 0$.
    \item \textbf{Upper Boundary ($S \to \infty$)}: $V(S, v, t) \approx S - K e^{-r(T-t)}$.
\end{enumerate}

\section{Architecture: Why KAN vs. MLP?}

\subsection{The Kolmogorov-Arnold Advantage}
Traditional MLPs rely on the Universal Approximation Theorem, which uses fixed activation functions on neurons. In contrast, KANs are based on the Kolmogorov-Arnold representation theorem:
\begin{equation}
    f(x_1, \dots, x_n) = \sum_{q=1}^{2n+1} \Phi_q \left( \sum_{p=1}^n \phi_{q,p}(x_p) \right)
\end{equation}
In a KAN, the activation functions live on the \textbf{edges} rather than the nodes. These functions $\phi_{q,p}$ are typically represented by B-splines.

\subsection{Superiority for PDEs}
\begin{itemize}
    \item \textbf{$C^2$ Continuity}: Solving second-order PDEs requires stable second-order derivatives. MLPs using ReLU are only $C^0$ continuous, leading to vanishing or non-existent second derivatives. KAN edges (B-splines or symbolic functions) are naturally $C^2$ or higher, providing smoother and more accurate PDE residuals.
    \item \textbf{Interpretability}: KANs can be reduced to symbolic mathematical expressions. This allows the \textbf{LiuClaw Agent} to inspect the model and replace learned splines with exact formulas like $\sin(x)$ or $\exp(x)$, improving generalization.
    \item \textbf{Sample Efficiency}: KANs often require significantly fewer parameters than MLPs to achieve the same accuracy in solving differential equations.
\end{itemize}

\section{The LiuClaw Agent: LLM reasoning in the Loop}

\subsection{Why LLM reasoning?}
While the KAN learns through gradient descent, it can become trapped in local minima or fail to identify global structural patterns (e.g., quadratic smiles). The LiuClaw Agent, powered by a high-reasoning LLM (e.g., DeepSeek-R1, Qwen 2.5), acts as a "Topological Surgeon":
\begin{itemize}
    \item \textbf{Mutation Logic}: Analyzes edge activation statistics and decides to \texttt{REPLACE} a B-spline with a symbolic function that better represents the underlying physics.
    \item \textbf{Regime Detection}: Uses Gaussian HMM outputs to adjust training priors. If a "Jump/Crash" regime is detected, the agent may mutate edges to favor heavier tails.
\end{itemize}

\subsection{Pydantic-Enforced DSL}
To ensure stability, the agent communicates via a strictly typed Domain-Specific Language (DSL). This prevents malformed commands from reaching the PyTorch execution environment.

\section{System Flow and Performance}

\subsection{Async Coordination Engine}
To maintain H200 throughput, the system uses a producer-consumer model:
\begin{enumerate}
    \item \textbf{Math Loop}: High-speed PyTorch training solving the PDE.
    \item \textbf{Reasoning Loop}: The LiuClaw agent evaluates the system state every $N$ iterations.
\end{enumerate}
This decoupled architecture ensures the H200 is never starved of compute while the LLM performs "slow" reasoning.

\subsection{Performance Benchmarks}
In our H200 stress tests, OpKAN achieved:
\begin{itemize}
    \item \textbf{Throughput}: $>26,000$ samples per second.
    \item \textbf{Convergence}: Stable loss decay from $\mathcal{O}(10^0)$ to $\mathcal{O}(10^{-3})$ within minutes.
\end{itemize}

\section{Conclusion}
OpKAN represents a paradigm shift in quantitative pricing. By embedding the Heston PDE into the topology of a Kolmogorov-Arnold Network and supervising its evolution with an LLM-driven agent, we achieve a level of interpretability and adaptability that was previously unattainable with numerical or standard deep learning methods.
